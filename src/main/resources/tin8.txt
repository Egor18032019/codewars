В Тинькофф решили освоить новую нишу, а именно открыть внутренний стартап по продаже доменов. Каждый домен представляет из себя строку, состоящую только из букв ﻿﻿SSS﻿﻿, ﻿﻿TTT﻿﻿, ﻿﻿AAA﻿﻿, ﻿﻿RRR﻿﻿.

Каждому покупателю хочется не просто продать строку, а продать нужную строку, а именно, пусть у каждого покупателя есть строки ﻿﻿PPP﻿﻿ и ﻿﻿QQQ﻿﻿. Тогда ему подходят только строки, у которых первые ﻿﻿∣P∣|P|∣P∣﻿﻿ символов совпадают с ﻿﻿PPP﻿﻿ и последние ﻿﻿∣Q∣|Q|∣Q∣﻿﻿ символов совпадают с ﻿﻿QQQ﻿﻿, т.е. все строки формата ﻿﻿P+A+QP+A+QP+A+Q﻿﻿, где ﻿﻿AAA﻿﻿ — некоторая строка.

Для начала было закуплено ﻿﻿nnn﻿﻿ доменов для ﻿﻿mmm﻿﻿ покупателей. Теперь хочется понять, хватит ли их для всех покупателей. Чтобы получить ответ на этот вопрос, посчитайте число подходящих для каждого покупателя строк.


Формат входных данных

Первая строка содержит два числа ﻿﻿nnn﻿﻿ и ﻿﻿mmm﻿﻿ ﻿﻿(1≤n≤105,1≤m≤105)(1 \leq n \leq 10^5, 1 \leq m \leq 10^5)(1≤n≤105,1≤m≤105)﻿﻿ — количество закупленных доменов и количество покупателей.

Следующие ﻿﻿nnn﻿﻿ строк содержат закупленные домены. В каждой строке встречаются только символы ﻿﻿SSS﻿﻿, ﻿﻿TTT﻿﻿, ﻿﻿AAA﻿﻿, ﻿﻿RRR﻿﻿. Длина каждой строки не более ﻿﻿10510^5105﻿﻿ символов, а их суммарная длина не превосходит ﻿﻿2⋅1062 \cdot 10^62⋅106﻿﻿.

Последние ﻿﻿mmm﻿﻿ строк содержат запросы покупателей. Каждый запрос задается двумя строками ﻿﻿PiP_iPi​﻿﻿ и ﻿﻿QiQ_iQi​﻿﻿, состоящими из вышеупомянутых символов. Длина каждой из строк ﻿﻿PiP_iPi​﻿﻿ и ﻿﻿QiQ_iQi​﻿﻿ не более ﻿﻿10510^5105﻿﻿символов, а суммарная длина всех строк ﻿﻿PiP_iPi​﻿﻿ не более ﻿﻿2⋅1062 \cdot 10^62⋅106﻿﻿. Аналогичное ограничение распространяется и на строки ﻿﻿QiQ_iQi​﻿﻿.


Формат выходных данных

Выведите ﻿﻿mmm﻿﻿ строк, состоящих из одного целого числа — количества подходящих строк для данных покупателей.
